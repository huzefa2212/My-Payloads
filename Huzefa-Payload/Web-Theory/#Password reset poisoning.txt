Password Reset Poisoning
-----------------------------------------------------------------------+

~ Password reset poisoning is a technique whereby an attacker manipulates a vulnerable website into generating a password reset link pointing to a domain under their control
~ This can arise due to a Host Header Injection Vulnerability
~ It can be used to steal password reset tokens
~ Even if the website uses the token only once , this vulnerability is still valid since we will be generating a new token


Basic Password Reset Poisoning -----------------------+

1) On attacker account request password reset link
2) perform full password reset of attacker account
3) Go to Burp HTTP History and look for a POST request for forgot-password , this will have the USERNAME or EMAIL parameter along with a CSRF token
4) Change the "host header" to the exploit server (burp collaborator) and change the USERNAME or EMAIL to victims'
5) go to burp collaborator and click poll now, copy the token and use the previous request URL(passsword reset URL from attackers email) but with new token
6) change password of victim account


Password Reset Poisoning Using Middleware -----------------------+

1) On attacker account request password reset link
2) perform full password reset of attacker account
3) Go to Burp HTTP History and look for a POST request for forgot-password , this will have the USERNAME or EMAIL parameter along with a CSRF token
4) add an "X-Forwarded-Host" header pointing to burp collaborator and change the USERNAME or EMAIL to victims'
5) go to burp collaborator and click poll now, copy the token and use the previous request URL(passsword reset URL from attackers email) but with new token
6) change password of victim account

2) perform full password reset of attacker account
3) set a protocol on the actual host and imbed a dangling markup payload with the src of attacker server
